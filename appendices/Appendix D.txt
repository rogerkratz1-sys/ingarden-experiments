Appendix D — Outlier Taxonomy (Full Classification Table)

Overview
Provides the complete, perpermutation classification table that underlies the outlier taxonomy summarized in Section 4.6 of the main paper. The appendix describes the table schema, the provenance and QC checks applied when generating the table, how the table is used to reproduce figures and statistics in Section 4.6, and where adjudication and stability artifacts are archived. The canonical CSV artifact is outputs_appendix_D/appendix_d_outliers_refined.csv and should be treated as the single source of truth for outlier reporting and exemplar retrieval.

Purpose and scope
* Records every permutation considered an outlier or exemplar in the analyses reported in Section 4.6.
* Makes explicit the structural, embedding, and provenance fields used to justify outlier status and motif assignments.
* Supports reproducible recreation of summary tables, figures, and manual adjudication workflows.

CSV schema (columns and oneline definitions)
ColumnDefinitionperm_indexUnique integer identifier for the permutation.perm_stringCommaseparated event index string representing the permutation.motif_primaryAdjudicated primary motif label (BRE, FM/EM, APD, DCO, GD, Other).motif_multilabelsSemicolonseparated multilabel flags satisfied by the permutation.kendall_normNormalized Kendall inversion distance to the canonical ordering.disp_kappa_scaledScaled (\kappa)-weighted displacement used in the hybrid metric.score_dirDirectional displacement scalar for the permutation.temporal_violationsCount of precedence inversions (temporal violations).scene_spanAggregate scene span metric for the permutation.speech_gapCount of speakercontinuity disruptions.disp_excessDisplacement beyond the tolerance window (w).embedding_delta_stressPerpoint change in stress after augmentation/refinement (embedding distortion).embedding_radiusEuclidean radius from the canonical region in the final 3D visualization.cluster_idClustering label used in motif discovery (DBSCAN/PAM).outlier_flagCategorical reason for outlier status (e.g., high_disp; high_inversions; geometric_artifact).stability_scoreFraction of labelsweep settings where the modal primary label occurs (0–1).short_notesOneline structural anomaly description.adjudicatorHuman adjudicator identifier if manual review occurred.provenance_commitGit commit hash used to generate the row.provenance_commandExact command or script invocation that produced the row.provenance_dateRun date/time (ISO 8601).checksumRowlevel checksum or pointer to artifact checksums for reproducibility.

How the table supports Section 4.6 and figures
* Reproducing summaries and plots: All aggregate statistics and exemplar selections in Section 4.6 are computed directly from this CSV. The exact aggregation commands used in the manuscript are recorded in scripts/analysis/summary_from_appendix_d.py.
* Exemplar traceability: Every figure exemplar references perm_index and provenance_commit so readers can retrieve the exact permutation, feature vector, and embedding coordinates.
* Adjudication transparency: Rows with nonempty adjudicator indicate manual review; the adjudication rubric and full logs are archived in Supplement S2.

Quality control and generation workflow
* Canonical generator: scripts/generate_appendix_d.py assembles feature vectors, multilabel flags, embedding diagnostics, and provenance into the CSV. The script emits a validation report and a provenance header.
* QC checks performed: no missing feature values; numeric ranges within expected bounds; consistency between motif_multilabels and motif_primary; checksum verification against checksums.md5. The QC script is scripts/validation/check_appendix_d_consistency.py.
* Provenance: each run embeds commit hash, exact command line, parameter file, and run timestamp in provenance_command/provenance_date. The definitive run used for the manuscript is recorded in the repository manifest.

Ambiguity, stability, and human validation flags
* Ambiguity flags: stability_score and motif_multilabels identify permutations with high multilabel overlap or low label stability; these rows are candidates for adjudication and are crosslisted in Supplement S8 (stability sweep outputs).
* Human validation set: a subset of rows used in the preregistered human evaluation are flagged in the CSV and linked to stimuli and pilot ratings in the supplement.
* Geometric artifacts: rows labeled geometric_artifact in outlier_flag point to counterexample permutations documented in Supplement S1.

Scripts, commands, and file locations (exact pointers)
* Table generator: scripts/generate_appendix_d.py --input outputs/metrics/all_features.csv --output outputs_appendix_D/appendix_d_outliers_refined.csv
* Summary reproduction: python scripts/analysis/summary_from_appendix_d.py --input outputs_appendix_D/appendix_d_outliers_refined.csv --out figures/section4_6/
* QC: python scripts/validation/check_appendix_d_consistency.py outputs_appendix_D/appendix_d_outliers_refined.csv
* Adjudication logs: supplement/S-2/adjudication_logs/
* Stability sweeps and threshold grids: supplement/S-8/diagnostics/
* Embedding diagnostics referenced by the table: outputs/embeddings/ (see augmentation_log.csv, nystrom_projection_log.csv, shepard_plot_data.csv)
Include the commit hash and exact command used for the manuscript run in the provenance header of the CSV.

Recommended usage notes for readers reusing the table
* Treat motif_primary as the descriptive, adjudicated label; consult motif_multilabels and stability_score before drawing strong inferences about ambiguous rows.
* Use provenance_commit and provenance_command to reproduce the exact environment and parameters.
* For reanalysis, run the QC script first to confirm local environment parity with the recorded checksums.
