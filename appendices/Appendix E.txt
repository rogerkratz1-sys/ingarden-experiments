Appendix E — Algorithmic and Implementation Details

Overview
Documents the permutationlevel metrics, deterministic decision rules, and implementation practices that drive motif assignment. This appendix supplies compact formulas, worked examples, pseudocode, and practical notes (file formats, BOM handling, key normalization, provenance) so readers can reproduce motif labels from the repository. Where relevant, Appendix E points to other appendices and supplements that contain tuning grids, adjudication logs, stability sweeps, and unit tests.

E.1 Base metrics (definitions and interpretation)
* Normalized Kendall distance (d_K(\pi,p_{\text{canon}})) — fraction of inverted event pairs relative to the canonical order:
[ d_K(\pi,p_{\text{canon}})=\frac{#{(i,j):i<j,\ \text{pos}\pi(i)>\text{pos}\pi(j)}}{n(n-1)/2}. ]
Interpretation: 0 = identical; 1 = fully reversed.
* (\kappa)-weighted scaled displacement (\mathrm{disp}_\kappa(\pi)) — perevent positional deviations aggregated with tunable sensitivity:
* \[
* \mathrm{disp}_{\kappa}(\pi)=\frac{1}{n}\sum_{e=1}^{n}\left|\mathrm{pos}_{\pi}(e)-p_{\mathrm{canon}}(e)\right|^{\kappa}
* \]
* \[
* \text{(optionally stabilized)}\quad
* \mathrm{disp}_{\kappa}^{\mathrm{scaled}}(\pi)=\mathrm{disp}_{\kappa}(\pi)\left(1+\alpha\cdot\mathrm{disp}_{\kappa}(\pi)^{2}\right)
* \]

* Interpretation: emphasizes large relocations when (\kappa>1).
* Anchor preservation — fraction of anchor events (\mathcal{A}) that remain within radius (r) of their canonical positions:
[ \text{anchor_pres}=\frac{1}{|\mathcal{A}|}\sum_{a\in\mathcal{A}}\mathbf{1}{|,\text{pos}\pi(a)-p{\text{canon}}(a),|\le r}. ]
* Max block move — size of the largest contiguous block in (\pi) that has been relocated as a unit relative to canonical order.
Interpretation: detects block reordering.
* Single element move — boolean or score indicating whether (\pi) can be obtained by moving a single element (or reversing a single insertion) from the canonical order.
Interpretation: distinguishes singleitem perturbations from block/global reorders.
* Orderdual proximity — Kendall distance between (\pi) and the reverse canonical ordering; low values indicate nearreverse permutations (used to detect DCO).
* Directional score score_dir — weighted linear aggregation of selected features:
[ \text{score_dir}(\pi)=\sum_f w_f\cdot\text{feature}_f(\pi), ] with default weights in config/weights.json (see Appendix B for weight rationale and sensitivity).

E.1 Worked example (compact)
Canonical (p_{\text{canon}}=[1,2,3,4,5,6]); (\pi=[1,3,2,4,6,5]).
* (d_K=2/15\approx0.133) (two inversions).
* Anchor preservation for anchors ({1,4}) with (r=0): 1.0.
* Max block = 1; single element move = false.
This illustrates how local adjacent swaps produce low (d_K) and low displacement relative to block/global motifs.

E.2 Motif assignment algorithm (pseudocode and rules)
Inputs and outputs
* Inputs: pi (permutation), p_canon, thresholds dictionary TH (JSON), auxiliary feature functions.
* Outputs: multilabel_flags (orderinvariant set), primary_label (precedenceresolved), features (provenance), adjudication_flag (manual review indicator).
Core pseudocode (condensed)
def motif_assignment(pi, p_canon, TH):
    # 1. compute features
    dk = kendall_distance(pi, p_canon)
    disp = disp_kappa_scaled(pi, kappa=TH['kappa'], alpha=TH['alpha'])
    anchor_pres = anchor_preservation(pi, anchors=TH['anchors'], r=TH['anchor_radius'])
    max_block = max_block_move(pi)
    single_move = single_element_move(pi)
    dual_prox = kendall_distance(pi, reverse_canonical(p_canon))
    score = score_dir(pi, weights=TH['weights'])

    # 2. order-invariant multilabel flags
    flags = set()
    if dk >= TH['GD_dK'] and disp >= TH['GD_disp']: flags.add('GD')
    if max_block >= TH['BRE_block'] and anchor_pres >= TH['BRE_anchor'] and disp >= TH['BRE_disp_min']: flags.add('BRE')
    if anchor_pres >= TH['APD_anchor'] and TH['APD_dK_low'] <= dk <= TH['APD_dK_high']: flags.add('APD')
    if single_move >= TH['FMEM_single'] and max_block < TH['BRE_block']: flags.add('FM/EM')
    if dual_prox <= TH['DCO_dual'] and score <= TH['DCO_score']: flags.add('DCO')

    # 3. primary label by precedence
    precedence = ['GD','FM/EM','BRE','DCO','APD','Other']
    primary = next((lbl for lbl in precedence if lbl in flags), 'Other')

    # 4. adjudication hook
    adjudicate = (len(flags) > 1) or (stability_fraction(pi) < TH.get('stability_threshold',0.8))

    return { 'multilabel_flags': sorted(flags), 'primary_label': primary, 'features': {...}, 'adjudication_flag': adjudicate }
Precedence and tiebreaking
Primary labels are chosen by a fixed precedence order (GD > FM/EM > BRE > DCO > APD > Other). Multilabel flags are preserved in the output and used in downstream stability reporting.

E.3 Implementation notes, determinism, and provenance
* Determinism: all computations are deterministic; any RNG used (e.g., for landmark selection) is seeded and recorded in provenance headers.
* Thresholds and weights: stored in versioned JSON files (thresholds.json, weights.json) and read at runtime. Each run records the exact TH file used in provenance_command.
* Batch processing: motif assignment is vectorized across permutations and exports order_invariant_multilabel_flags.csv, labels_per_perm.csv, and adjudication_candidates.csv.
* Adjudication workflow: adjudication candidates include perm_index, multilabel_flags, features, stability_fraction, and a freetext adjudicator_note; logs are archived in Supplement S2.

E.4 File formats, BOM handling, and key normalization (robustness fixes)
* Problem: JSON files authored with a UTF8 BOM or Pythonstyle mappings caused loader failures and KeyError when keys were strings vs integers.
* Fixes implemented: 
o BOMsafe loader: open JSON with encoding='utf-8-sig' to strip leading BOM.
o Fallback parser: if JSON parsing fails, attempt ast.literal_eval to accept Pythonstyle mappings.
o Key normalization: convert keys to integers immediately after loading: p_canon = {int(k): v for k,v in p_canon.items()}.
o Tolerant lookups: metric functions use p_canon.get(key, p_canon.get(str(key))) to avoid KeyError during transition.
* Files changed: scripts/motif_assignment.py (loader/normalizer), scripts/compute_metrics.py (tolerant lookups).
* Recommended practice: keep canonical JSON with consistent key types; include small sample inputs in tests/ for CI.

E.5 Stability, adjudication, and testing
* Stability threshold: default stability_threshold = 0.8; permutations below this are flagged for adjudication. Stability sweeps and the focused threshold grid are documented in Appendix I and archived in Supplement S8.
* Unit tests: recommended pytest cases for identity, single swap, and reversed permutations for kendall_distance and disp_kappa_scaled. Test harnesses and sample inputs live in tests/.
* Duplicate header cleanup: postprocessing step removes repeated header lines from CSV exports when present.

E.7 Practical guidance for readers and reusers
* Inspect motif_multilabels and stability_score before treating motif_primary as definitive.
* Use provenance_commit and provenance_command to recreate the exact environment; run the QC script before reanalysis.
* For CI and future edits, add unit tests for any change touching kendall_distance, disp_kappa_scaled, or the loader.
