Appendix I — Type I Error Diagnostic and Label Robustness

Overview
This appendix documents (A) the Type I error diagnostic used to test whether clusters found in the peripheral region of the SMACOF embedding can arise under a conservative radialpreserving null, and (B) the labelrobustness sweep that quantifies how primary motif labels respond to plausible threshold perturbations and precedence permutations. The appendix gives procedural detail, exact commands, pseudocode, pvalue conventions, runtime notes, and pointers to the full numeric outputs and persetting diagnostics in Supplement S8.

Type I Error Diagnostic — Methods
Goal. Test whether clusters discovered in the peripheral subset produce clusterdensity statistics plausibly generated by a radialpreserving null.
Pipeline summary
* Peripheral selection: points with Euclidean radius ? the (p)th percentile (definitive runs used p = 95; sensitivity runs at p = 85, 90, 95).
* Clustering: DBSCAN on the peripheral subset with eps = 0.5, min_samples = 5. Each nonnoise DBSCAN label is a candidate motif.
* Test statistic: cluster density (T_k = n_k / A_k) where (n_k) is cluster size and (A_k) is cluster area (convex hull area in 2D projection; PCA projection fallback available). See S8_methods_equations.txt for formulas.
* Null model: radialpreserving resampling (--null_method radial_preserve) that preserves marginal radii while randomizing angular positions.
* Monte Carlo: (B = 1000) null replicates for reported runs (increase (B) for more precise smallp estimates).
* Multiple testing: Benjamini–Hochberg FDR at (\alpha = 0.05).
* Reporting: raw Monte Carlo (\hat p) and BH q; null quantiles and summaries are in S8_null_samples_summary.csv.
Sanity checks
* Inspect percandidate null histograms and quantiles.
* Confirm DBSCAN parameters produce expected candidate counts on the peripheral subset.
* Run smallB sanity checks (e.g., (B=200)) before full (B=1000) runs.




Label Robustness — Methods
Motivation. Motif assignment depends on numeric thresholds and precedence rules. We quantify stability of primary labels under plausible threshold perturbations and precedence permutations.
Twostage sweep
1. Stage 1 (OAT): oneatatime scan across numeric thresholds to identify sensitive keys (43 OAT settings).
2. Stage 2 (focused grid): exhaustively vary the top three sensitive keys — disp_kappa_scaled, max_block_move, order_dual_proximity — across multipliers ([0.5,0.75,0.9,1.0,1.1,1.25,1.5]) (343 focused settings).
Settings and data
* Total settings: 386 (43 OAT + 343 focused).
* Permutations evaluated: 5,000.
* Long table: diagnostics/stability_map.csv (one row per permutation per setting; 5,000 × 386 = 1,930,000 rows).
* Summary: diagnostics/stability_summary.csv (one row per permutation: modal_label, modal_fraction, stability_score, multilabel_union).
Primary label selection and stability score
* Compute orderinvariant multilabel flags per permutation and setting.
* Select primary label by fixed precedence (reported runs used: BRE, FM/EM, APD, DCO, GD, Other).
* Stability score = fraction of settings where the modal primary label occurs (range 0–1).
Precedence ensemble
* Sampled 50 random precedence orders and recomputed primary labels to assess tiebreaking sensitivity.

Results — Type I Error (numeric)
* Definitive run: p = 95, B = 1000. Wall time for the full Monte Carlo run ? 1,166.5 s on the reported environment.
* Observed candidates (example canonical run): 
o Candidate 0: size = 227, observed statistic (T_{\mathrm{obs}} \approx 5.387\times 10^{6}), Monte Carlo (\hat p \approx 0.000999).
o Candidate 1: size = 17, Monte Carlo (\hat p \approx 1.0).
* Multiple testing (BH): see motif_results/candidates.csv and S8_null_histograms.txt for percandidate (\hat p) and BH q. (All perrun candidate tables and the merged sensitivity table sensitivity_table_p85_90_95.csv are in Supplement S8.)
* Injection power: injected tight Gaussian clusters (example: (n_{\text{inject}}=50); 50 trials) produced empirical detection_rate = 1.00 for inject_size = 50 under DBSCAN eps = 0.5, min_samples = 5 (see S8_injection_power_curves.txt).
* Stability under jitter: mean Jaccard ? 1.0 for the reported clusters (see S8_stability_matrix.txt).
Interpretation: the pipeline reliably detects compact injected clusters of the tested sizes; perrun candidate statistics and BH decisions are recorded in Supplement S8. See the candidate CSVs in motif_results_robustness/peripheral_* for the authoritative perrun decisions and the provenance manifest for each run.

Results — Label Robustness (numeric)
* Permutations analyzed: 5,000.
* Settings evaluated: 386.
* Precedence sensitivity: 0 permutations changed primary label across the 50 sampled precedence orders.
* Worst observed stability: no permutation had stability_score < 0.5; the lowest modal fraction among the 50 least stable permutations ? 0.7409.
* Union of everobserved flags (across settings): Other: 5000; APD: 1581; FM/EM: 294; BRE: 23.
* Flip patterns: flips are few, structured, and concentrated in the focused grid (typical flips: Other ? FM/EM, Other ? APD, Other ? BRE).
* Interpretation: multilabel overlap is common, but primary labels (precedence + thresholds) are highly stable under the explored perturbations; tiebreaking order is not a major driver of instability in the sampled ensemble.
