Supplement S2 Exemplar Diagnostics and Pilot Reader Ratings

This supplement provides the pilot reader ratings, perexemplar diagnostics, and minimal analysis artifacts that directly support the preregistered humanevaluation plan described in Appendix G. It contains deidentified raw responses, aggregated summaries, exemplar diagnostics, plotting scripts, and a manifest of provenance headers and checksums so readers can reproduce the pilot analyses and link pilot estimates to the preregistered power calculations and exclusion rules.

Files included and purpose
* supplement/reader_ratings_sample_all6.csv — deidentified raw pilot responses (columns: participant_code, exemplar_id, coherence_rating, anchor_preserved, reading_time_ms, attention_check_passed).
* supplement/reader_ratings_summary.csv — aggregated summaries used for pilot reporting (mean_rating, sd_rating, n_readers, anchor_preservation_rate).
* supplement/diagnostics_per_exemplar.csv — perexemplar diagnostics used to select stimuli (permutation_vector, cover_violation_count, kendall_tau, anchor_preservation_flag, stability_fraction, structural_metric, short_notes).
* supplement/diagnostics_summary.csv — merged diagnostics and rating summaries used for Table S1 comparisons.
* supplement/comparison_table.csv — structural versus perceptual ranking table (Table S1).
* supplement/plots/onepanel_figure.png — bar figure of mean coherence ratings by exemplar.
* supplement/plots/rating_vs_structural_metric.png — scatterplot of mean rating versus structural metric.
* supplement/scripts/make_onepanel_figure.py — plotting script that reads reader_ratings_summary.csv and writes the figures.
* supplement/README_S2.txt — manifest with exact generation commands, provenance checksums, and the commit SHA used to produce these artifacts.

Methods summary and relation to Appendix G
* Design — Pilot participants rated all six exemplars of the Ten Virgins parable on a 7point coherence scale and answered an anchorpreservation question; presentation order was randomized. This mirrors the withinsubjects design specified in Appendix G.
* Primary outcome — 7point coherence rating (1 = least coherent, 7 = most coherent), identical to Appendix G.
* Exclusion rules — Trials failing attention checks or with reading times < 200 ms or > 3 median absolute deviations above the participant median were excluded, consistent with Appendix G preprocessing rules.
* Use of pilot data — Pilot estimates were used to parameterize power simulations archived under supplement/S-6/power/ and to inform covariate selection and robustness checks in Appendix G.

Pilot results snapshot
* Key pilot correlations — mean_rating versus structural_metric: (\rho = -0.657) (p = 0.156). mean_rating versus cover_violation_count: (\rho = -0.829) (p = 0.0416).
* Interpretation — In this pilot, coverviolation counts tracked perceived incoherence more closely than the hybrid structural metric; this informed the preregistered decision to include coverviolation measures and stability flags as covariates and moderators in Appendix G’s analysis plan.

Reproducibility and provenance
* Provenance headers — Each CSV/JSON in this supplement begins with a provenance header containing git_commit=<short-sha>, generated_for=Appendix G preregistered human evaluation, command=<exact command used>, and an ISO8601 UTC timestamp. Example: 
* # provenance: git_commit=9a809e5
* # generated_for: Appendix G preregistered human evaluation
* # command: python scripts/generate_reader_summaries.py --input raw.csv --output reader_ratings_summary.csv
* # timestamp: 2026-03-02T19:13:00Z
* Environment — supplement/requirements.txt lists the minimal packages (pandas, seaborn, matplotlib) required to run the plotting script.
* Checksums — supplement/manifest_checksums.txt contains SHA256 checksums for each artifact and the commit SHA used to generate them.
* Reproduction steps — From the repository root: 
1. python -m pip install -r supplement/requirements.txt
2. python supplement/scripts/make_onepanel_figure.py
3. Confirm outputs in supplement/plots/.

Data sharing, ethics, and editorial note
* Privacy — Raw pilot CSVs are deidentified; participant_code is a nonreversible token. Any requests for reidentifiable data must follow the consent and datasharing procedures documented in Appendix G.
